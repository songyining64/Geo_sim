# Meta-AMG 实验实施计划

---

## 一、数据集：从哪来

### 来源一：MatrixSequenceGenerator（主力，仅需CPU）

这是已实现完成的合成数据生成器，模拟非线性Picard迭代中粘度场演化导致的矩阵序列变化。可控参数如下。

| 参数 | 可选值 | 论文使用的值 |
|------|--------|------------|
| 矩阵规模 | 64-5000 DOF | 100, 225, 400, 625, 900, 1600, 2500 |
| 粘度对比度 | 10² - 10⁶ | 10², 10³, 10⁴, 10⁵, 10⁶ |
| 地质模式 | slab, plume, layered, random | 全部四种 |
| 序列长度 | 4-15步 | 8步（训练）, 10步（测试） |
| 训练序列数 | 100-2000 | **500条**（产生4000个task） |

```
运行命令:
  python experiments/run_experiments.py --exp all

数据生成耗时估算:
  500条序列 × 每条8步 × 每步矩阵生成+AMG标注 ≈ 每条3秒
  总计: 约25分钟 (纯CPU)
```

### 来源二：Underworld实际仿真数据（可选，需安装Underworld）

用于验证Meta-AMG在真实Stokes矩阵上的泛化能力。不需要大量数据，20-50条序列作为测试集即可。

提取方式：修改Underworld在每个Picard迭代步输出当前刚度矩阵（CSR格式的`.npz`文件），然后用我们的`AMGDataGenerator.compute_ground_truth_cf()`标注C/F。

### 来源三：标准稀疏矩阵数据集（可选，用于泛化测试）

SuiteSparse矩阵库中的结构力学/热传导类矩阵，可在论文中作为"非地质场景的泛化测试"。

## 二、训练方案

### 2.1 完整训练（论文用）

```
训练规模:
  训练序列: 500条 × 平均8步 = 4000个相邻矩阵对(task)
  验证序列: 100条 × 平均8步 = 800个task
  
训练参数:
  GNN: 3层 GCNConv, hidden=64
  MAML: inner_lr=0.01, outer_lr=0.001, inner_steps=3
  Epochs: 50
  Batch: meta_batch_size=4

计算资源:
  GPU (单卡): 预计48小时
  CPU (16核): 预计120小时（不推荐但可行）
```

### 2.2 快速验证训练（调试用）

```
训练序列: 50条, Epochs: 10
耗时: GPU约1小时，用于验证代码和初步看趋势
```

### 2.3 训练数据生成细节

每条训练序列模拟一次完整的非线性Picard迭代：“均匀粘度场 → 逐渐演化到目标粘度场”。四类目标场分述如下。

slab模式模拟冷俯冲板片：在计算域中对角线带状区域施加高粘度（η_max = contrast），周围为正常粘度，模拟冷的刚性板块插入软流圈。plume模式模拟热地幔柱：中心圆形区域为低粘度（η_min = 1/contrast），周围为正常粘度，模拟深部热物质上涌。layered模式模拟岩石圈-软流圈分层：上部30%区域为高粘度（刚性岩石圈），下部为正常粘度（软流圈）。random模式产生随机块状粘度分布，用于测试模型对不规则地质结构的泛化能力。

## 三、六个实验的具体规格

### E1：收敛性验证

目的是证明Meta-AMG预测的C/F能产生与传统AMG收敛到相同解的V-cycle。对每个测试矩阵规模（100到2500中的四个档位），先生成一条slab模式序列，然后分别用传统AMG和Meta-AMG求解序列中的每个矩阵。记录相对误差，并重复三次取均值。

预期结果：所有规模下相对误差低于10⁻⁵，随着矩阵增大不会出现精度退化。预期表格如下。

| 矩阵规模 | 传统AMG误差 | Meta-AMG误差 | 相对差异 |
|---------|-----------|-------------|---------|
| 225 | ~10⁻¹⁶ | ~10⁻⁵ | 可忽略 |
| 625 | ~10⁻¹⁶ | ~10⁻⁵ | 可忽略 |
| 1600 | ~10⁻¹⁶ | ~10⁻⁵ | 可忽略 |

### E2：Setup加速比

这是论文最核心的实验。在七个矩阵规模（100到2500）上，每个规模生成一条slab模式序列，分别记录传统AMG和Meta-AMG的单步平均setup时间，重复五次。表格列目为矩阵规模、传统AMG平均时间和标准差、Meta-AMG平均时间和标准差以及加速比。

预期结果：矩阵越大加速比越大，交叉点约在200自由度附近。n=100时Meta-AMG比传统慢（GNN固定开销约20毫秒超过传统AMG的亚毫秒级setup），但从n=400开始加速比快速爬升至10倍以上，到n=2500时可望超过50倍。

### E3：消融实验

分成三个子实验。消融3a对比零样本预测和MAML适配后的C/F准确率，预期适配后比零样本高15到20个百分点，这一差距随着训练任务数增加而扩大。消融3b测试不同适配步数（1、2、3、5、10步）的效果，预期3步附近达到饱和——更少步数导致适配不充分，更多步数带来边际收益递减和线性增加的时间开销。消融3c对比一阶MAML（不计算二阶梯度）和二阶MAML的效果差异，预期二阶在训练任务较少时明显更优，因为二阶梯度提供了更精确的元梯度方向，但当任务数充足时两者差距缩小。

### E4：粘度对比鲁棒性

在400自由度的slab模式矩阵序列上，变化粘度对比度（10²、10³、10⁴、10⁵、10⁶），每个对比度生成五条序列取平均。预期结果：随着对比度增大，零样本预测的准确率逐步下降（因为高对比度矩阵在训练数据中占比少），但MAML适配后的准确率保持相对稳定。传统AMG在高对比度下因为条件数恶化可能需要更多V-cycle迭代才能收敛，而Meta-AMG通过GNN学到的局部粗化策略恰好适应了这种稀疏模式的变化。

### E5：可扩展性

在225自由度的矩阵上训练，在更大的测试规模（400到2500四个档位）上测试泛化能力。每个测试规模生成slab模式序列，记录零样本和适配后的C/F准确率以及求解误差。预期结果：训练和测试规模差距在10倍以内时，适配准确率损失不超过10个百分点；差距超过10倍时性能逐渐退化，但仍优于随机猜测。这个实验直接验证了GNN的规模泛化能力，是论文中最有说服力的实验之一。

### E6：方法对比

在同一矩阵序列上运行三种方法：每步都重新setup的传统AMG（baseline）、每步独立预测的神经AMG（已有方法的代表）、以及从第二步起用MAML适配的Meta-AMG（本文方法）。记录每步的setup时间和求解迭代次数。预期结果：神经AMG的setup比传统快但与Meta-AMG持平或略慢（因为它每次需要完整的GNN推理而非增量适配），而Meta-AMG在从第二步开始的序列累积加速中显著领先。求解迭代次数三者应当持平，因为都使用相同的插值公式构建P算子。

## 四、预期结果总表

| 实验 | 核心指标 | 预期值 | 是否达标的标准 |
|------|---------|--------|-------------|
| E1 收敛性 | 相对误差 vs 传统AMG | < 10⁻⁵ | 所有规模通过 |
| E2 加速比 | T_trad / T_meta | n=100时 <1x, n=400时 >10x, n=2500时 >50x | 大矩阵显著加速 |
| E3a 消融 | 适配 vs 零样本准确率 | 适配高 15-20 百分点 | 差距 > 10 百分点 |
| E3b 步数 | 最佳适配步数 | 3步接近饱和 | 3步 vs 1步差距明显, 5步 vs 3步差距小 |
| E3c 阶数 | 一阶 vs 二阶 MAML | 少任务时二阶更好 | 与理论一致 |
| E4 对比度 | 准确率 vs contrast | 高对比度下适配保持 >80% | 10⁶时不低于0.75 |
| E5 规模泛化 | 小训大测的准确率 | 10x差距内损失 <10 百分点 | n=2500时 > 70% |
| E6 方法对比 | Meta vs Neural vs Trad | Meta总时间最短 | 序列越长Meta优势越大 |

## 五、总时间估算

按阶段展开的工时如下。

训练数据生成准备和调试大约需要6小时，生成完整的500条训练序列和验证序列。完整训练耗时约48小时（GPU）或者120小时（CPU），训练过程中只需要监控loss曲线和验证准确率即可，大部分时间可以并行做其他工作。六个实验的运行在训练完成后只需约3小时即可输出全部数据。绘图脚本编写需要约4小时生成论文所需的五张图——加速比折线图、消融柱状图、对比度鲁棒性图、规模泛化图和收敛性验证图各一张。

总计：从零开始到拿到全部可以写入论文的实验数据和图表，GPU条件下约需60小时（两天半），CPU条件下约需130小时（五天半）。实际工作日考虑调试和反复，建议预留一周。
