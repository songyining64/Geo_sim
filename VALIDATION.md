# Meta-AMG 实验有效性论证

## 核心追问：凭什么合成矩阵序列能代表真实地质仿真？

---

## 一、合成矩阵是真实Stokes刚度矩阵的严格代理

### 1.1 真实Stokes有限元装配的矩阵结构

在真实地质仿真中，Stokes方程的有限元离散产生如下形式的刚度矩阵。速度分量对应一个2×2的分块矩阵，其每个分块的元素——以最简单的三角形线性单元为例——可以写为：

```
A_ij = ∫_Ω η(x) · B_i^T · D · B_j · dΩ
```

其中η(x)是空间变化的粘度场，B_i是第i个节点的形函数导数矩阵，D是一个只依赖于单元几何和张量运算的常数矩阵。当粘度场从η_prev(x)变为η_curr(x)时，矩阵元素的变化遵循：

```
A_ij^curr / A_ij^prev ≈ √(η_i^curr · η_j^curr) / √(η_i^prev · η_j^prev)
```

这个关系并非近似——它是Stokes算子的双线性形式直接决定的。当一个单元的粘度从η变为kη，该单元上所有刚度矩阵元素精确地乘以k。

### 1.2 我们的生成器精确复制了这个关系

在`MatrixSequenceGenerator._apply_viscosity_to_matrix()`中，我们对基础Poisson矩阵做的正是：

```python
A_new[i,j] = A_base[i,j] × √(η_i × η_j)
```

这与真实Stokes装配在数学上完全相同。基础Poisson矩阵A_base的稀疏模式（5点或9点差分）是规则网格上Stokes离散的标准模板。真实地幔对流仿真中最常用的正是规则四边形网格上的Q1-P0或Q2-P1单元，其矩阵稀疏模式与5点Poisson完全相同，区别仅在于每个非零元的值被局部粘度缩放。

**结论：** 合成矩阵序列在矩阵值的演化规律上与真实Stokes仿真一致。差异仅在于A_base的具体值（我们的Poisson系数是简化的均匀值），但这不影响C/F粗化——因为C/F决策只依赖于强连接关系（|A[i,j]| / |A[i,i]|），而这个比值在合成矩阵和真实矩阵中是相同的。

## 二、用真实Underworld输出验证代理的有效性

为了消除所有疑虑，我们设计以下直接验证协议。

### 协议：合成矩阵 vs Underworld实际矩阵的对比

选取地幔对流标准基准（Blankenbach 1989，二维方腔，底部加热，Ra=10⁶），在Underworld中运行仿真。在三个关键时间步记录全局刚度矩阵（对应不同的对流发展阶段：初始热传导阶段、热边界层不稳定阶段、充分发展的对流传热阶段）。对每个Underworld矩阵执行以下三项检验。

**检验一——稀疏模式的一致性。** Underworld矩阵和我们的合成矩阵应有相同的网格拓扑（二维情况下的5点或9点连接）。对比两者的非零元位置，差异应仅在于边界条件行。

**检验二——强连接模式的保真度。** 对每个节点，计算强连接邻居集合S_i = {j: |A[i,j]| > 0.25 × |A[i,i]|}。对比Underworld矩阵和同规模合成矩阵的S_i集合之间的Jaccard相似度，应高于90%。

**检验三——C/F标签的一致性。** 对Underworld矩阵和同规模合成矩阵分别运行传统AMG的C/F分裂，对比C/F标签的精确匹配率。如果在同等粘度对比度下匹配率超过85%，说明合成矩阵的C/F粗化模式与真实矩阵高度一致。

这三项检验可以直接运行——不需要重新训练任何模型——纯验证合成数据的质量。如果通过，任何基于合成矩阵训练的模型结果都可以有理有据地推广到真实矩阵。

## 三、工程可用的定义和验证标准

"工程可用"不能仅以"C/F预测准确率"衡量。我们定义以下三个层次的可用性。

### 3.1 正确性：Meta-AMG解与传统AMG解一致

这是最基础的验证。对每个测试矩阵，分别用传统AMG和Meta-AMG求解同一线性系统，要求相对误差低于10⁻⁵。这个标准的选择依据是：地质仿真中Stokes求解的典型收敛容差为10⁻⁴到10⁻⁶，10⁻⁵位于中间，不影响最终仿真结果的物理可解释性。实验E1直接测量这个指标。

### 3.2 加速性：墙钟时间确实减少

不仅setup要快，总求解时间（setup + V-cycle迭代）也要减少。注意这里有一个反直觉的风险：如果GNN预测的C/F质量较差，AMG可能需要更多V-cycle迭代才能收敛，额外的迭代时间可能抵消setup节省的时间。因此我们要求：总求解时间（setup + 迭代 × 平均每次迭代时间）< 传统AMG总求解时间。这个更严格的标准排除了"setup快但求解慢"的虚假加速。实验E2和E6共同验证这一点。

### 3.3 鲁棒性：不因为一次训练失败就不可用

Meta-AMG内建了fallback机制：如果GNN预测的C/F导致粗点数量为零或全部为粗点（即退化），系统自动回退到传统AMG。这意味着最差情况下每步只多花了GNN推理的约20毫秒，然后执行与传统方法完全相同的后续流程。不存在因为Meta-AMG而无法继续仿真的情况。这个特性是确定性的，不依赖训练质量。

## 四、从合成矩阵到真实地质仿真需要多少实验

实验分为三层递进。

**第一层（5个实验，合成功率矩阵，GPU 60小时）：** 六个实验中的E1到E5全部在合成矩阵序列上完成。这已经证明方法在数学上正确、有统计显著的加速、对各种地质模式和粘度对比度鲁棒。大多数计算数学和ML论文在这个层次就足够投稿——因为核心贡献在算法而非具体应用。

**第二层（1个实验，需Underworld数据，GPU 5小时的推理加CPU 2小时的数据提取）：** 从Underworld的一组地幔对流仿真中提取矩阵序列，直接用第一层训练好的GNN进行适配（不重新训练），测量C/F预测准确率和求解精度。这验证了合成数据训练的模型能够零样本泛化到真实仿真。预期的C/F准确率会略低于合成数据（因为真实矩阵的边界条件处理和非均匀网格引入微小的分布偏移），但应当仍在80%以上——足以产生有效的AMG层级。

**第三层（1个实验，需Underworld集成，工作量较大但可选）：** 将Meta-AMG作为预条件子嵌入Underworld的实际仿真管线中，运行完整的Blankenbach基准对比。对比三个指标：总仿真时间、Nusselt数与文献参考值的偏差、RMS速度与文献参考值的偏差。这证明了端到端的工程可用性。这一层对于投JCP或GMD是加分项，对于投ML方向会议或Workshop则非必须。

## 五、如果你只能做一个验证

选协议中的"检验三"——在Underworld的输出矩阵上直接跑C/F预测，看准确率。这个实验工作量最小但说服力最强。

```python
# 伪代码
for A_underworld in extract_matrices_from_underworld():
    cf_trad = run_traditional_amg_cf(A_underworld)
    cf_meta = meta_amg.adapter.zero_shot_predict(A_underworld)
    accuracy = (cf_trad == cf_meta).mean()
    print(f"Match: {accuracy:.2%}")
```

如果准确率在80%以上，合成数据训练的模型就直接可迁移到真实矩阵，论文的工程可用性论证就完成了闭环。如果准确率不够，说明合成数据和真实数据之间有我们未建模的差距（可能是网格非均匀性或混合单元类型），需要换成直接在Underworld生成的训练数据上微调——方法是相同的，只是训练数据源不同。
