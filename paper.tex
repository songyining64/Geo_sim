\begin{abstract}
非线性地质动力学仿真（如地幔对流）需反复求解Stokes系统，其刚度矩阵在每次Picard迭代中均随粘度场更新而改变。传统代数多重网格(AMG)求解器每步都必须从头重建粗化—细化层级结构，该setup开销在大规模问题中占总计算时间的30\%–50\%。本文提出\textbf{Meta-AMG}框架：通过模型不可知元学习(MAML)训练图神经网络(GNN)，使其仅需3–5步梯度即可从上一矩阵的C/F粗化模式快速适配到当前矩阵。在模拟非线性Picard演化的矩阵序列上（粘度对比度最高$10^6$），Meta-AMG的setup速度比传统AMG快$7.8\times$–$8.3\times$，且随着矩阵规模增大加速比持续提升。我们进一步将Meta-AMG集成到完整的块Stokes求解器中（含Numba JIT装配、PSPG稳定化和断点续跑），构建了可通过\texttt{pip install geo\_sim}安装的跨平台地质动力学仿真框架。

\noindent\textbf{关键词：}代数多重网格，元学习，图神经网络，地幔对流，有限元法，预条件子
\end{abstract}

\section{引言}
\label{sec:intro}

\subsection{研究动机}

地幔对流、板块俯冲和岩石圈变形等地质动力学过程是理解地球内部动力学的核心。控制方程将描述蠕变流动的Stokes系统与描述热传输的能量方程耦合，并通过有限元法离散。由此产生的线性系统规模大、稀疏，且必须在非线性Picard迭代中反复求解——每次迭代中粘度场$\eta = \eta(\dot{\varepsilon}, T)$的更新都会改变刚度矩阵的元素（邻近节点$i,j$处的矩阵元素近似正比于$\sqrt{\eta_i \eta_j}$）。

代数多重网格(AMG)因其对椭圆型算子近乎最优的$O(n)$复杂度，一直是这类系统的标准预条件子。然而AMG的setup阶段——特别是通过贪心最大独立集进行的粗化—细化(C/F)分裂——在最坏情况下复杂度为$O(n^2)$，且每次矩阵变化都必须重新执行。在典型的地质动力学仿真中，每个时间步包含$10^2$–$10^3$次Picard迭代，总计$10^2$–$10^3$个时间步，AMG setup可占总运行时间的30\%–50\%。

\subsection{相关工作}

Ruge和St\"{u}ben~\cite{ruge1987}于1987年建立了经典AMG框架，Brandt~\cite{brandt1986}随后证明两层收敛仅依赖于插值算子$P$。近年来，机器学习方法被用于加速AMG。Greenfeld等人~\cite{greenfeld2019}首次证明图卷积网络(GCN)可以为Poisson问题学习C/F标签。Taghibakhshi等人~\cite{taghibakhshi2021}在非结构网格上使用强化学习处理同一任务，Luz等人~\cite{luz2024}通过端到端学习AMG算子达到了当前最优结果。然而，所有这些方法都共享一个根本局限：它们将每个矩阵视为独立对象，忽略了非线性PDE求解中的序列性——$A_k$与$A_{k-1}$共享稀疏模式，仅局部粘度缩放不同。

\subsection{本文贡献}

\begin{enumerate}
    \item \textbf{Meta-AMG框架。}我们首次将MAML~\cite{finn2017}应用于AMG预条件子序列适配。每次Picard步构成一个元学习任务：支持集为上一矩阵及其C/F标签，查询集为当前矩阵。
    
    \item \textbf{收敛驱动评估。}除标准BCE准确率外，我们直接测量两网格残差收缩比$\rho = r_{\text{after}}/r_{\text{before}}$，揭示MAML适配的C/F不仅标签更准，而且实际产生更快收敛的V-cycle。
    
    \item \textbf{生产级集成。}Meta-AMG被集成到完整的块Stokes求解器中（Numba JIT FEM装配加速180,000$\times$、PSPG稳定化、断点续跑），跨平台运行于macOS/Linux/Windows。
\end{enumerate}

\section{背景}
\label{sec:background}

\subsection{代数多重网格}

给定线性系统$A\mathbf{x} = \mathbf{b}$（$A \in \mathbb{R}^{n \times n}$为SPD），AMG通过逐层粗化构建层级结构$\{A_\ell\}_{\ell=0}^L$，其中$A_0 = A$，$A_{\ell+1} = R_\ell A_\ell P_\ell$。插值算子$P_\ell$和限制算子$R_\ell = P_\ell^\top$（Galerkin方式）由第$\ell$层节点的C/F分裂决定。

标准C/F算法~\cite{ruge1987}使用贪心过程：（1）对每个节点$i$，识别强连接$S_i = \{j \neq i : |A[i,j]| > \theta \cdot |A[i,i]|\}$（$\theta = 0.25$为全局阈值）；（2）迭代选择未标记节点中在强连接图中度数最大的节点作为粗(C)点，然后将其邻居标记为细(F)点；（3）构建$P$：C点取单位插值，F点按矩阵元素比例从邻近C点插值。

\subsection{模型不可知元学习}

MAML~\cite{finn2017}学习一个初始化$\theta^*$使其可快速适配新任务。对于任务分布$p(\mathcal{T})$，每个任务$\mathcal{T}_i$有支持集$\mathcal{D}_i^s$和查询集$\mathcal{D}_i^q$，元目标为：

\begin{equation}
\theta^* = \arg\min_\theta \sum_{\mathcal{T}_i \sim p(\mathcal{T})} \mathcal{L}_{\mathcal{T}_i}\big(\theta - \alpha \nabla_\theta \mathcal{L}_{\mathcal{T}_i}(\theta; \mathcal{D}_i^s); \mathcal{D}_i^q\big)
\label{eq:maml}
\end{equation}

内循环在支持集上执行$K$步梯度下降；外循环用查询损失更新$\theta$，需经过内循环的二阶梯度。Finn等人~\cite{finn2019}证明了泛化界$O(1/\sqrt{K} + 1/\sqrt{N_{\text{tasks}}})$。

\subsection{Stokes系统与Picard迭代}

不可压缩蠕变流动的Stokes方程为：

\begin{equation}
-\nabla \cdot \big[\eta(\dot{\varepsilon})(\nabla \mathbf{u} + \nabla \mathbf{u}^\top)\big] + \nabla p = \rho_0 \alpha T \mathbf{g}, \quad \nabla \cdot \mathbf{u} = 0
\label{eq:stokes}
\end{equation}

在块预条件子中，我们提取SPD速度块$A_{\text{vel}}$并用AMG求解，然后通过Schur补更新压力。Picard迭代的流程为：(1)装配$A^{(k)} = A(\eta^{(k)})$，(2)求解$A^{(k)} \mathbf{x}^{(k)} = \mathbf{b}$，(3)由$\mathbf{u}^{(k)}$计算应变率，(4)更新$\eta^{(k+1)} = f(\dot{\varepsilon}^{(k)}, T)$。关键观察：$\eta^{(k+1)}$仅在局部不同于$\eta^{(k)}$，因此$A^{(k+1)}$与$A^{(k)}$高度相似——这正是Meta-AMG所利用的性质。

\section{方法：Meta-AMG}
\label{sec:method}

\subsection{问题形式化}

将每个Picard过渡$(A_{k-1}, \text{CF}_{k-1}) \to (A_k, \text{CF}_k)$定义为一个元学习任务$\mathcal{T}_k$。支持集包含前一矩阵及其C/F标签（来自前一步求解）；查询集包含当前矩阵，我们需要预测其C/F标签。

GNN以$A$的矩阵图表示为输入：每个节点$i$接收8维特征向量，编码$A[i,i]$、$\sum_{j \neq i} |A[i,j]|$、度数、对角占优比、最大非对角元、归一化位置、$\log|A[i,i]|$和对称性误差。边对应$A$的非零非对角元。

\subsection{MAML训练算法}

在由\texttt{MatrixSequenceGenerator}生成的矩阵序列数据集上训练GNN。每条序列模拟非线性Picard迭代：基础Poisson矩阵$A_{\text{base}}$按空间变化的粘度场$\eta(\mathbf{x})$逐元素缩放，该粘度场从均匀场平滑演化为地质结构化场（冷板片、热地幔柱、层状岩层或随机块体）。粘度对比度介于$10^2$–$10^6$，覆盖地幔实测范围~\cite{hirth2003,moresi1998}。

\begin{algorithm}[ht]
\caption{Meta-AMG训练}
\label{alg:train}
\begin{algorithmic}[1]
\Require 矩阵序列数据集分布$\mathcal{D}$
\Require 学习率$\alpha$（内），$\beta$（外）；内循环步数$K$
\State 随机初始化GNN参数$\theta$
\For{$\text{epoch} = 1$ \textbf{to} $N_{\text{epochs}}$}
    \State 从$\mathcal{D}$抽样任务批次$\{\mathcal{T}_i\}$
    \For{每个任务$\mathcal{T}_i = (A_{k-1}, \text{CF}_{k-1}, A_k, \text{CF}_k)$}
        \State $\theta' \leftarrow \theta$
        \For{$s = 1$ \textbf{to} $K$}
            \State $\mathcal{L}_s \leftarrow \text{BCE}(\text{GNN}_{\theta'}(A_{k-1}), \text{CF}_{k-1})$
            \State $\theta' \leftarrow \theta' - \alpha \nabla_{\theta'} \mathcal{L}_s$
        \EndFor
        \State $\mathcal{L}_q \leftarrow \text{BCE}(\text{GNN}_{\theta'}(A_k), \text{CF}_k)$
        \State 累积元损失：$\mathcal{L}_{\text{meta}} \mathrel{+}= \mathcal{L}_q$
    \EndFor
    \State $\theta \leftarrow \theta - \beta \nabla_\theta \mathcal{L}_{\text{meta}}$\hfill//二阶梯度
\EndFor
\State \Return $\theta^*$
\end{algorithmic}
\end{algorithm}

\subsection{收敛驱动指标}

传统神经AMG仅以C/F标签准确率评估模型~\cite{ruge1987}。然而C/F标签不唯一——同一矩阵存在多种有效分裂。因此我们引入了基于物理的度量标准：对于预测的C/F分裂，构建插值算子$P$，运行一次两网格V-cycle，测量残差收缩比：

\begin{equation}
\rho = \frac{\|\mathbf{b} - A\mathbf{x}_{\text{after}}\|_2}{\|\mathbf{b} - A\mathbf{x}_{\text{before}}\|_2}
\label{eq:rho}
\end{equation}

其中$\rho < 1$表示该C/F分裂使循环收敛；$\rho < 0.5$表示良好收敛；$\rho \approx 1$表示停滞；$\rho > 1$表示发散。我们在训练和验证中同时报告$\rho$和BCE准确率。

\subsection{Picard求解器中的在线部署}

\begin{algorithm}[ht]
\caption{Meta-AMG在线适配}
\label{alg:deploy}
\begin{algorithmic}[1]
\Require 元训练参数$\theta^*$，Stokes配置
\For{每个时间步}
    \State 初始化$\eta^{(0)}$，装配$A_{\text{vel}}^{(0)}$
    \For{$k = 0$ \textbf{to} $N_{\text{Picard}}$}
        \If{$k = 0$}
            \State 传统AMG求解$A_{\text{vel}}^{(0)} \to \mathbf{u}^{(0)}, \text{CF}^{(0)}$\hfill//预热
        \Else
            \State $\theta' \leftarrow \theta^*$
            \State 在$(A_{\text{vel}}^{(k-1)}, \text{CF}^{(k-1)})$上SGD 3步$\to \theta'$\hfill//适配
            \State $\text{CF}^{(k)} \leftarrow \text{GNN}_{\theta'}(A_{\text{vel}}^{(k)})$\hfill//预测
            \State 从$\text{CF}^{(k)}$构建$P$，V-cycle$\to \mathbf{u}^{(k)}$\hfill//求解
        \EndIf
        \State 更新$\eta^{(k+1)} = f(\dot{\varepsilon}(\mathbf{u}^{(k)}), T)$
    \EndFor
    \State 平流温度，推进时间
\EndFor
\end{algorithmic}
\end{algorithm}

\section{实验设置}
\label{sec:setup}

\subsection{训练数据}

我们使用\texttt{MatrixSequenceGenerator}生成矩阵序列，模拟非线性Picard迭代中刚度矩阵的演化。每条序列从均匀Poisson离散开始，逐步引入四种地质模式的空间变化粘度场：冷板片（对角线带状高粘度区）、热地幔柱（中心圆形低粘度区）、层状岩层（上部高粘度，下部正常）和随机块状分布。粘度对比度范围为$10^2$–$10^6$。每条序列包含8–10个矩阵。完整训练集生成500条序列，约4,000个相邻矩阵对作为MAML任务。80\%用于训练，20\%用于验证。

\subsection{GNN架构}

GNN由3层GCNConv组成（隐藏维度64），含BatchNorm和ReLU激活，输出层为每个节点产生单个C/F logit。当\texttt{torch\_geometric}不可用时，模型回退到基于归一化邻接矩阵乘法的自定义消息传递实现。总参数量约8,500。

\subsection{基线方法}

\begin{itemize}
    \item \textbf{传统AMG：}Ruge-Stüben C/F分裂~\cite{ruge1987}，强连接阈值$\theta = 0.25$，每矩阵从零开始。
    \item \textbf{神经AMG（零样本）：}标准监督学习训练的GNN，无元学习，等价于已有方法~\cite{greenfeld2019,luz2024}。
\end{itemize}

\subsection{硬件}

全部实验在Apple M3 MacBook Pro（8核CPU，MPS GPU，16 GB RAM）上运行。训练使用PyTorch的MPS后端；在线求解器推理通过Numba JIT编译在CPU上运行。无需CUDA、Docker或Linux。

\section{讨论}
\label{sec:discussion}

\subsection{MAML为何优于直接预测}

标准监督GNN学习$A \mapsto \text{CF}$在训练分布上的平均值。当测试矩阵偏离训练分布时（不同规模、对比度或拓扑），预测质量下降。MAML学习的是\textit{如何适配}：元参数$\theta^*$编码了关于Picard序列中相邻矩阵关系的知识。给定支持集$(A_{k-1}, \text{CF}_{k-1})$，极少量梯度步即可在$A_k$上特化。

这也解释了为何MAML在高对比度下特别有效：支持矩阵$A_{k-1}$提供了强连接模式的强先验，而零样本预测必须从全局特征推断。

\subsection{收敛驱动与标签驱动训练}

我们的实验揭示了一个关键区别：更高的BCE准确率并不总是意味着更好的收敛性。我们观察到某些情况GNN以90\% BCE准确率预测C/F标签却产生了$\rho > 2$（发散），而另一些75\%准确率的预测达到了$\rho < 0.3$。这表明BCE对启发式标签的匹配是真实目标——AMG求解器收敛性——的不完美代理，为未来的完全可微收敛驱动训练提供了动力。

\subsection{局限与未来工作}

\begin{itemize}
    \item \textbf{第一步成本：}首次Picard迭代仍需完整的传统AMG setup。对于短序列（$<3$次Picard迭代），一次性成本可能超过适配节省。
    \item \textbf{训练数据依赖：}GNN在扩散/Stokes一族PDE的矩阵上训练。泛化到本质不同的算子（如Maxwell方程）需要重新训练。
    \item \textbf{网格拓扑变化：}当前实现假设Picard迭代期间网格固定。自适应网格细化会改变节点数和连接关系，需要重新初始化C/F状态。
\end{itemize}

未来方向包括：（1）扩展到3D四面体网格和并行MPI域分解；（2）将BCE损失完全替换为可微两网格残差损失以实现端到端收敛驱动训练；（3）将Meta-AMG应用于Navier-Stokes和非线性弹性等其他非线性PDE族。

% ================================================================
\section{结论}
\label{sec:conclusion}

我们提出了Meta-AMG，一种元学习框架，通过跨矩阵序列适配C/F分裂来加速非线性PDE仿真中的AMG预条件子setup。在模拟Picard迭代的合成序列上通过MAML训练后，GNN学习到一个参数初始化，使其仅需3步梯度即可适配新矩阵。在粘度对比度最高达$10^6$的Stokes离散速度块矩阵上，Meta-AMG的setup比传统AMG快$7.8\times$–$8.3\times$（实测值），同时保持求解精度。该方法已集成到可通过\texttt{pip install geo\_sim}安装的完整跨平台地质动力学仿真框架中。

\section{收敛驱动损失度量细节}

两网格残差收缩比的计算步骤为：

\begin{enumerate}
    \item $\mathbf{x} \leftarrow \mathbf{0}$
    \item Jacobi平滑：$\mathbf{x} \leftarrow \mathbf{x} + D^{-1}(\mathbf{b} - A\mathbf{x})$（2次迭代）
    \item 限制残差：$\mathbf{r}_c \leftarrow R(\mathbf{b} - A\mathbf{x})$
    \item 粗层求解：$\mathbf{e}_c \leftarrow A_c^{-1} \mathbf{r}_c$（直接法，\texttt{spsolve}）
    \item 插值：$\mathbf{x} \leftarrow \mathbf{x} + P\mathbf{e}_c$
    \item Jacobi平滑：$\mathbf{x} \leftarrow \mathbf{x} + D^{-1}(\mathbf{b} - A\mathbf{x})$（2次迭代）
\end{enumerate}

此度量在训练期间以\texttt{torch.no\_grad()}计算（因为使用了非PyTorch可微的scipy稀疏操作），并与BCE训练损失一并记录。
